# Relatório de Execução — 2026-05-02

## 🎯 Objetivo da Sessão
Otimização de infraestrutura do ecossistema Ywara para viabilizar operação em VPS de baixo custo (2GB RAM) e migração definitiva para Logto Cloud.

---

## 🏗️ Decisões Técnicas e Arquiteturais

### 1. Migração para Logto Cloud (Modelo Híbrido)
- **Decisão:** Substituição do Logto Self-Hosted (containers locais + banco Postgres) pelo **Logto Cloud (SaaS)**.
- **Impacto:** Redução imediata de ~700MB de RAM ociosa. O gerenciamento de identidades agora é externo, simplificando o `docker-compose.yml` e eliminando o custo de manutenção do banco de dados de autenticação.

### 2. Otimização de Recursos (VPS 2GB)
- **Memory Limits:** Aplicados limites estritos de RAM para todos os serviços Node.js e MySQL:
    - `kurupira-backend`: 512MB (CPU-bound)
    - `nexus-db` (MySQL 8.0): 512MB
    - `iaca-backend`: 256MB
    - `neonorte-admin-backend`: 256MB
    - `api-gateway`: 64MB
- **MySQL Tuning:** Criação de `my.cnf` customizado com `innodb_buffer_pool_size = 256M` e desabilitação do `performance_schema`, garantindo estabilidade em ambientes de baixa memória.

## ⚡ Estabilização do Catálogo FV (PVSyst V7 Support)

### 🎯 Objetivo
Resolver inconsistências de dados e falhas de ingestão no Catálogo de Equipamentos do Sumaúma, garantindo paridade técnica total com o PVSyst.

### 🏗️ Decisões Técnicas e Arquiteturais

#### 1. Universalização do Parser PVSyst
- **Correção Crítica:** Identificado que o backend (`panOndParser.js`) possuía uma lógica de pop de stack (`>`) divergente do frontend (`>=`). Isso causava a "perda" de parâmetros elétricos que ficavam presos dentro de sub-objetos comerciais.
- **Normalização:** Alinhamento dos níveis de indentação entre front e back (`level = Math.floor(indent / 2)`), garantindo que arquivos com diferentes espaçamentos sejam interpretados corretamente.

#### 2. Suporte Nativo ao PVSyst V7 (DMEGC, Longi, Trina)
- **Conversão Automática de Unidades:** Implementada lógica de detecção de magnitude para coeficientes térmicos. Valores em `mV/°C` (comuns em V7) são convertidos automaticamente para `% / °C` usando a tensão nominal do módulo.
- **Aliases de Engenharia:** Adicionado suporte a chaves variáveis do PVSyst:
    - Potência: `PNom`, `Pnom`, `Pmax`.
    - Coeficientes: `muVocSpec`, `muIsc`, `muPmpReq`.
    - Térmico: `Tnom`, `T_NOCT`.

#### 3. Refatoração do Motor de Validação
- **Poka-Yoke de Tipagem:** Ajuste no `equipmentValidator.js` para tratar `null` de forma estrita. Corrigido o bug silencioso onde `null >= 0` resultava em `true` no JS, disparando alertas de erro críticos para campos ausentes.
- **Extração Limpa:** O campo `electricalData` (JSON) agora é sanitizado antes da persistência, removendo metadados brutos do PVSyst e mantendo apenas as métricas de engenharia normalizadas.

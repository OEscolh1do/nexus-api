# 🗺️ Framework de Prompt Engineering para Desenvolvimento de Software

> **🎯 Objetivo:** Transformar desenvolvimento assistido por IA de "Vibe Coding" (improvisação) para **Engenharia de Precisão**.

---

## 🚀 Para Quem é Este Framework?

- ✅ Desenvolvedores que querem **extrair o máximo** de IAs como ChatGPT, Claude, Gemini
- ✅ Pessoas com **TDAH** que precisam de estrutura para manter o foco
- ✅ Equipes que querem **padronizar** o uso de IA no desenvolvimento
- ✅ Estudantes aprendendo **boas práticas** de engenharia de software

---

## 🛠️ Agnóstico de Stack

Este framework funciona com **qualquer** tecnologia:

- **Frontend:** React, Vue, Angular, Svelte, Next.js, etc.
- **Backend:** Node.js, Python, Java, Go, Ruby, PHP, etc.
- **Database:** SQL (PostgreSQL, MySQL), NoSQL (MongoDB, Redis), etc.

### 📝 Configuração Inicial

1. Copie [`STACK_CONFIG.md`](./STACK_CONFIG.md) e preencha com sua stack tecnológica
2. Os templates se adaptarão automaticamente aos seus placeholders
3. Comece a usar os templates com contexto específico do seu projeto

## 🔍 L - O Detetive de Prompts

**Novo!** Agente especialista em transformar intenções vagas em prompts estruturados através de dedução lógica.

- [**L - Detetive de Prompts**](./L/) - Análise probabilística, perguntas estratégicas, quirky

**Características:**

- 🧠 Analítico (vê padrões invisíveis)
- 📊 Probabilístico (calcula chances de sucesso)
- 🍰 Quirky (analogias com doces)
- 🔍 Dedutivo (nunca assume, sempre deduz)

**Como funciona:**

1. Você: "Quero fazer um app" (vago)
2. L: Faz 5 perguntas estratégicas
3. L: Gera prompt XML estruturado
4. Resultado: 91% de chance de sucesso (vs 8% com prompt vago)

**Quando usar:**

- Você tem ideia vaga
- Precisa de análise lógica
- Quer aprender a pensar em prompts
- Gosta de probabilidades e dedução

**Quick Start:** [L/QUICK_START.md](./L/QUICK_START.md)

## 🎓 Filosofia

### O Problema: "Vibe Coding"

Pedir para a IA "criar uma feature" sem contexto → Código caótico, bugs, retrabalho.

### A Solução: Engenharia Estruturada

1. **Planejar** (Template 01: Arquiteto)
2. **Executar** (Template 02: Engenheiro)
3. **Verificar** (Template 03: Testes)
